# Mines Game 測試個案集 (Test Case Suite)

本文件根據 `proposal.md` 定義之功能需求，規劃詳細的測試個案，確保遊戲邏輯、賠率計算、公平性及安全性符合設計規範。

---

## 1. 核心演算法測試 (Core Algorithm)
驗證賠率計算引擎的正負準確性，防止浮點數誤差及邏輯錯誤。

| ID | 測試標題 | 測試步驟 | 預期結果 | 優先級 |
|:---:|:---|:---|:---|:---:|
| TC-001 | 組合數學 C(n, r) 驗證 | 輸入 n=25, r=5 等邊界值。 | 輸出應精確等於 53130 (使用 BigInt 避免溢位)。 | P0 |
| TC-002 | 賠率 (Multiplier) 精準度測試 | 輸入地雷數 5, 已揭開格子數 1, RTP 96%。 | 倍率應精確顯示為 1.200x (對照表參考值)。 | P0 |
| TC-003 | RTP 邊界值驗證 | 測試 RTP 設定為 94%, 96%, 97%, 98%, 99%。 | 系統應能正確讀取並應用對應係數，不接受範圍外數值。 | P1 |
| TC-004 | 浮點數誤差處理 | 連續進行 24 次安全點擊 (24 顆雷情境)。 | 最終倍率計算不應出現 `NaN` 或怪異的長小數，應四捨五入至特定位數。 | P1 |

## 3. 可證明公平性測試 (Provably Fair)
驗證系統不具備作弊可能性，且玩家可反向驗證。

| ID | 測試標題 | 測試步驟 | 預期結果 | 優先級 |
|:---:|:---|:---|:---|:---:|
| TC-201 | Seed Hash 匹配驗證 | 遊戲開始取得 Hash，結束取得 Seed。 | `SHA256(Seed)` 必須與開局給出的 Hash 完全一致。 | P0 |
| TC-202 | 地雷位置隨機性驗證 | 使用相同 Seed 模擬多次洗牌。 | 只要 Seed 相同，洗牌結果 (Fisher-Yates) 必須恆定且符合公開算法。 | P1 |
| TC-203 | Client Seed 影響測試 | 修改玩家提供的 Client Seed。 | 地雷位置應隨 Client Seed 改變而不同，確保後端無法單方面操控。 | P2 |

## 5. 安全性與異常處理 (Security)
防止作弊與系統崩潰。

| ID | 測試標題 | 測試步驟 | 預期結果 | 優先級 |
|:---:|:---|:---|:---|:---:|
| TC-401 | 重複點擊測試 (Double Pick) | 對同一格子發送兩次 `/game/pick`。 | 第二次應報錯或被忽略，不影響倍率計算。 | P1 |
| TC-402 | 越權 Session 操作 | 使用 A 玩家的 sessionId 嘗試替 B 玩家點擊。 | 回傳 403 Forbidden 或 404 Not Found。 | P0 |

| TC-404 | 伺服器異常回復 | 在 Pick 過程中強制重啟後端服務。 | 若使用 Redis 存儲則可恢復，若使用記憶體則應優雅處理退回下注或紀錄損失。 | P2 |
